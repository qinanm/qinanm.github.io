提升用户体验篇

主要关于用户体验的提升，从四个方面入手讲解了如何在不同的方面来提升用户的体验

1. 页面展示方面

1. 布局方面

  1. 整体布局

    可以分为全屏布局和定宽布局，定宽布局的话一般使用margin：0 auto; 使页面保持居中

    目前更常使用的使用左边定宽，右边自适应，使用flex布局，父元素设置flex, 左边设置flex-basis为宽度大小确定所占的宽度位置，右边设置flex-grow为1自适应放大

  2. footer布局

    要求：在页面高度小于屏幕高度的时候，footer显示在最底部

    有两种实现手法

    1. margin-top:auto

    2. flex布局，column主轴，justify-content：space-between

2. 文本方面

防御性编程：所有外部数据都是不可信的

从后端接口获取的数据，要做各种预设，考虑全面。

1. 文本超长

设置超出部分隐藏，并将边界部分文本设为省略号显示

针对按钮内的文本

内容较少时，设置最小宽度限制，min-width

内容过多时，设置padding

如上图所示即可

3. 0内容展示

情境： 当搜索一个内容没有搜索到结果的时候，即无数据的时候，页面需要展示的内容

0 内容页面的设计，需要起到对用户的引导作用，让用户知道数据为空的原因是什么，而不是一片空白的等用户去猜

首先要判断出现无数据的原因是什么

  • 数据为空：用户无权限、搜索无结果、筛选无结果、页面无数据
  • 异常状态：网络异常、服务器异常、加载失败

3. 图片方面

1. 给图片同时设置宽高

在创建img标签的时候就给它设置固定的宽高，在图片未加载前就提前占住位置，以避免后续插入图片的时候引起重排消耗性能

给插入的图片也设置同等的宽高

2. 设置object-fit

限制了图片的宽高就很可能出现图片拉伸的问题。

利用object-fit: cover; 就能使图片在保持其宽高比的同时填充元素的整个内容

它还有一个配套属性object-position: 50% 50%可以控制图片在内容框的位置

3. 图片加载失败

利用图片加载失败，触发<img>的 onerror事件，给加载失败的<img>新增一个样式类

利用新增的样式类，配合<img>伪元素，展示默认兜底的图片和<img>的alt信息

2. 交互设计优化

1. 过渡与动画

在交互过程中，适当的增加过渡和动画，让用户感知页面的变化

2. 滚动优化

  • 滚动平滑：使用 scroll-behavior: smooth 让滚动丝滑
  • 使用 scroll-snap-type 优化滚动效果
  • 控制滚动层级，避免页面大量重排

3. 点击交互优化

在用户点击交互中，加一些有意思的细节

  • 按钮可点击: cursor: pointer
  • 按钮禁止点击：cursor: not-allowed
  • 等待 Loading 状态：cursor: wait
  • 输入框：cursor: text;
  • 图片查看器可放大可缩小：cursor: zoom-in/ zoom-out
  • 提示：cursor: help;

1. 点击区域优化

伪元素扩大点击区域

2. 快速选择优化

user-select: all

3. 选择样式优化

：：selection

CSS 提供一个 ::selection 伪类，可以控制选中的文本的样式（只能控制color, background, text-shadow），进一步加深效果。

4. 添加禁止选择

user-select: none
