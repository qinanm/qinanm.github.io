hexo 搭建博客

1 hexo是什么

博客框架，使用markdown语法，一键编译生成静态文件，一键部署到github或其他等代码托管平台

2 前期安装准备

  1. node.js

    • 官网https://nodejs.org/en/
    • 验证 ： node -v

  2. git

    • 官网 https://git-scm.com/download/win
    • 验证： git -v

  3. cnpm

    • npm install -g cnpm –registry==https://registry.npm.taobao.org

    • 验证：cnpm -v

  4. hexo

    • cnpm install -g hexo-cil
    • 验证 hexo -v

  5. 保存自动刷新，安装插件

    • npm install -g browser-sync
    • 验证： browser-sync –version
    • 在根目录下执行 npm install hexo-browsersync –save
    • hexo s 测试
      • 目的：修改 Markdown 文件并保存后浏览器可以自动刷新
      • 不过我修改配置到是没啥效果，可能只能用在markdown文件上了

8 关于hexo内显示图片的问题

我自己有一个解决的想法，就是把本地图片放在source文件夹的mdImgs文件夹中

然后显示图片的方式

后面我感觉太麻烦了，就去百度

于是有了新的解决办法

方法1：添加图片，先npm install 一个hexo-asset-image的依赖

使用这个插件后，就是可以在每次创建一个md文件的时候自动创建一个同名的文件夹，同名的文件夹中可以存放图片

方法2： 搭建阿里云图床 + typora配置实现对象存储

方法来自https://blog.csdn.net/weixin_42030522/article/details/123075824

  1. 首先申请一个对象存储OSS，登录阿里云 –>进入工作台 –>对象存储OSS

  2. 点击创建Bucket — 这里注意读写权限那里，一开始选了公共读，后面报403错误了,后面将它修改成了公共读写才可以正常使用

  3. typroa安装PicGo-Core插件

    文件 —> 偏好设置

    1. 修改配置文件

      在偏好设置里打开配置文件

    2. 找到配置文件内必要键对应的值

      accessKeyId

      accessKeySecret

      这两个键通过accessKey 管理 — 创建子用户 — 创建成功后记下对应的accessKeyId和accessKeySecret填入配置文件即可

      其他必要的键对应的值

    3. 在typora偏好设置里验证图片上传 ，不产生错误即成功了，如果产生错误得话，去阿里云查看对应的错误码和错误描述得到解决或者自己看错误提示来微调解决
