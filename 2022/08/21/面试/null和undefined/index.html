null和undefined

发表于2022-08-21|更新于2022-08-21|js
|阅读量:

在面试过程中经常会被问到的问题就是null和undefined的区别，最初我并没有太在意，每次都是回答null就是空，undefined就是未定义，但是从面试官的反应来看好像总不是太满意，于是今天就具体了解了一下两者的区别。

事实上null和undefined都可以表示 无 的意思，那么为什么js既要有null又要有undefined呢, 在 阮一峰的博客undefined与null的区别中其实已经将历史原因说的很具体了。

简要概括一下就是，JavaScript的设计者Brendan Eich认为（1）表示 无 的值最好不是一个对象，而null属于空对象； （2）初代js没有错误处理机制，Number(null)自动转换为0，不容易发现错误。 于是就有了undefined.

那么关于这两者的差别具体有

  1. 概念上

    null表示“没有对象”，即此处不该有值

    undefined表示“缺少值”， 就是此处应该有值，但是还没有定义

  2. 使用上

    null

    • 作为函数的参数，表示该函数的参数不是对象

    • 作为对象原型链的终点

    undefined

    • 变量声明了，但是没有赋值
    • 调用函数时，应该提供的参数没有提供
    • 对象没有赋值的属性
    • 函数没有返回值时，默认返回undefined

  3. 内置判断上

    • Number(null) //0; Number(undefined) //NaN;
    • typeof null //Object; typeof undefined // undefined;

总结：

null和undefined的区别我认为最重要的就是概念上，null表示此处不该有值，而undefined表示此处应该有值但是还未定义，开发者只要明确这一点就可以更清楚应该使用null还是undefined了。
