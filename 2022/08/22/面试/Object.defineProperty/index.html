Object.defineProperty

发表于2022-08-22|更新于2022-08-22|js
|阅读量:

起源

对于Object.defineProperty()就我熟知的就是用来定义一个对象的属性，然而这个功能仅通过obj.key = value就可以实现，那么为何又到大费周章的使用Object.definedProperty()呢，原因就是通过Object.definedProperty定义的属性可以给它设置一些特性.

参数

Object.defineProperty的参数有

  • obj: 对象
  • prop: 对象的属性（可以新增，可以修改）
  • descriptor: 描述符，定义属性特性，对象形式

由此可见，descriptor是Object.defineProperty中最重要的属性，当然这三个属性也缺一不可。

descriptor

descriptor可以以两种描述符形式存在，一种是数据描述符，一种是存取描述符，它内部存在有一些键值对。

数据描述符和存取描述符共有的键值对

  • configurable 表示可配置的。当值为true时，该属性的描述符才可以被改变，同时该属性也能从对应的对象中删除
  • enumerable 表示可枚举的。当值为true时，该属性才会在对象枚举时枚举到

笔试小插曲

数据描述符独有的

  • value : 表示属性对应的值。
  • writable: 表示可写的，只有值为true时，属性值才能被修改。(默认值false,对于使用Object.defineProperty()新增的属性有效，但是对于对象中原本就有的属性进行修改是无效的，原本的属性依旧可以修改)

存取描述符独有的

  • set: 当对象的属性修改时调用set函数，参数为修改的值
  • get: 当读取对象的属性时调用get函数，无参数

默认值

  • configurable、enumerable、writable三者的默认值是false;
  • value、set、get三者的默认值是undefined;

注意

  • 数据描述符和存储描述符都有的键值对不能同时存在
  • 当没写任一描述符的独有键值对时，默认是数据描述符
